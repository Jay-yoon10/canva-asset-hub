What it does

Built for real enterprise workflows

Large marketing teams manage thousands of approved assets in S3. This eliminates the manual middle step.

⚡

Event-driven, zero manual work

S3 PutObject fires EventBridge, which triggers Lambda, which calls Canva — all without a human touching anything.

🤖

AI tagging that actually means something

Amazon Bedrock generates business-context tags like brand_tier: premium and approved_for: social_media — not just "beach" or "sunset".

🔄

It goes both ways

Canva Webhooks push finished designs back to S3. The loop closes automatically — from raw asset to published design and back.

🪣 S3 Trigger

  • PutObject event fires on image upload
  • EventBridge rule filters by prefix & suffix
  • Supports PNG, JPG, SVG

λ Lambda Handler

  • Generates presigned URL for S3 object
  • Initiates Canva async upload job
  • Polls until status = success

🎨 Canva Integration

  • OAuth 2.0 PKCE token flow
  • Async upload + job polling pattern
  • Asset lands in Canva Brand Kit

📊 Observability

  • CloudWatch structured logging
  • Success / failure metrics
  • Lambda duration tracking

🤖 Bedrock Tagging

  • Claude Model
  • brand_tier, campaign_type, approved_for
  • Business context — not just visual labels

🔄 Reverse Sync

  • DESIGN_PUBLISH webhook event
  • Design exported → saved to S3
  • HMAC signature validation

🗄 Data Layer

  • DynamoDB: metadata + sync state
  • GSI on canva_asset_id
  • TTL for cost control

🔐 Auth & UI

  • Amazon Cognito User Pools
  • JWT authoriser on API Gateway
  • React dashboard: status + logs

Endpoints

Backend REST API (API Gateway + Lambda) plus key Canva Connect API endpoints.

✅ Phase 1 — Implemented · Canva Connect API (S3 → Canva Pipeline)
POST /v1/asset-uploads Initiate async asset upload (returns job ID) ▾
Auth — OAuth 2.0 PKCE · Scope: asset:write
HeaderValue
AuthorizationBearer {access_token}
Asset-Upload-Metadata JSON with name_base64 — filename Base64-encoded
e.g. {"name_base64": "b2NlYW5fMS5wbmc="}
Content-Typeapplication/octet-stream
Example Response
{
    "job": {
      "id": "ded634f4-650c-4a16-8317-06ae058731fd",
      "status": "in_progress"
    }
}
GET /v1/asset-uploads/{job_id} Poll upload job until complete ▾
Polling Pattern (Python — Lambda)
# Poll every 3s until status == "success" (max 10 attempts)
  for attempt in range(10):
      time.sleep(3)
      res = canva_get(f"/v1/asset-uploads/{job_id}")
      status = res["job"]["status"]
      if status == "success":
          asset_id = res["job"]["asset"]["id"]
          break
      elif status == "failed":
          raise Exception("Upload failed")
